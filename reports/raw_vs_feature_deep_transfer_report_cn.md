# 原始数据与特征数据迁移训练说明报告

## 1. 结论先行

本项目中的深度域适配训练不是直接使用原始轴承振动波形或飞轮原始遥测序列训练，而是先从原始数据中提取具有物理意义的退化特征，再将这些特征输入深度编码器进行迁移学习。

当前训练路径为：

```text
XJTU-SY 轴承原始 CSV
-> 时域/频域/时频域特征
-> Bearing Encoder
-> z_b

飞轮附件 1/2 原始数据
-> 归一化电流/温度/转速/HI/HI 导数等特征
-> Flywheel Encoder
-> z_f

z_b 与 z_f
-> CORAL / MMD / DANN / AutoEncoder-joint 域适配训练
-> RUL 误差验证
```

因此，深度迁移训练使用的是“从原始数据提取后的退化特征”，不是完全端到端的原始波形训练。

## 2. 为什么不直接使用原始数据

轴承数据和飞轮数据在模态与时间尺度上差异很大：

- XJTU-SY 轴承数据是高频振动信号，单个 CSV 内包含大量振动采样点。
- 飞轮附件数据是天级遥测数据，主要变量为电流、温度、转速等慢变状态量。
- 轴承数据描述的是地面加速寿命试验，飞轮数据描述的是在轨长期退化过程。

如果直接使用原始数据训练，模型需要同时解决三个问题：

1. 高频振动波形与低频遥测序列的采样尺度不一致。
2. 轴承分钟级寿命试验与飞轮天级寿命过程的时间尺度不一致。
3. 两类数据传感物理含义不同，原始信号空间无法直接对齐。

在当前样本量和数模任务约束下，直接端到端迁移容易出现训练不稳定、可解释性弱、结果难以写入论文的问题。

## 3. 使用特征训练的合理性

本项目选择先进行特征工程，再进行深度迁移训练，原因如下：

| 对比项 | 原始数据训练 | 特征数据训练 |
|---|---|---|
| 输入形式 | 原始振动波形、原始遥测序列 | RMS、Kurtosis、FFT energy、Wavelet、STFT、HI 等 |
| 信息量 | 更完整 | 有一定压缩 |
| 稳定性 | 较低 | 较高 |
| 可解释性 | 较弱 | 较强 |
| 跨模态对齐难度 | 很高 | 明显降低 |
| 数模论文适配性 | 较弱 | 较强 |

特征训练并不是“偷懒”，而是将不同模态的原始信号先转换为统一的退化语义空间。例如：

- 轴承振动中的 RMS、谱能量、峭度等反映冲击增强和故障发展。
- 飞轮电流、温度、HI 等反映摩擦增强和性能退化。
- 两者在物理层面都可以对应到“退化程度增加”这一共同状态。

因此，将特征输入编码器后再对齐 latent space，比直接对齐原始信号更符合本题的工程背景。

## 4. 当前深度迁移实验是否完整

当前项目已经完成了真实 PyTorch 深度迁移训练，而不是只有接口。

已训练方法包括：

- `no_adaptation`：无域适配基线。
- `CORAL`：协方差对齐训练。
- `MMD`：最大均值差异最小化训练。
- `DANN`：梯度反转和域判别器对抗训练。
- `AutoEncoder-joint`：跨域联合重构与 latent 对齐训练。

验证方式为：

- 使用附件 1 已知全寿命数据。
- 前 70% 作为训练段。
- 后 30% 作为真实 RUL 测试段。
- 比较迁移前后 RUL 误差。

核心结果：

| 方法 | A1 测试 MAE / 天 | A1 测试 RMSE / 天 | A2 参考 RUL / 天 |
|---|---:|---:|---:|
| MMD | 87.2 | 101.6 | 408.5 |
| AutoEncoder-joint | 117.1 | 141.7 | 377.4 |
| no_adaptation | 154.2 | 174.0 | 119.0 |
| DANN | 167.7 | 199.0 | 6200.0 |
| CORAL | 200.0 | 213.4 | 4.9 |

结果说明，MMD 域适配相比无迁移基线降低了 RUL 测试误差，证明轴承退化特征对飞轮退化建模具有可迁移价值。

## 5. 与完全端到端原始数据训练的区别

完全端到端原始数据训练通常需要：

- 对轴承原始振动波形使用 CNN、1D-CNN、Transformer 或时频图网络。
- 对飞轮遥测序列使用 LSTM、TCN 或 Transformer。
- 再设计跨模态对齐模块，将两个不同采样尺度的序列映射到统一 latent space。
- 需要更多训练样本、更长调参时间，以及更严格的验证集。

当前项目没有采用这种路线。当前路线是：

```text
原始数据
-> 物理退化特征
-> 深度编码器
-> latent 对齐
-> RUL 验证
```

这属于“特征级深度迁移学习”，不是“原始信号级端到端迁移学习”。

## 6. 对论文表述的建议

论文中建议这样写：

> 鉴于轴承振动信号与飞轮遥测数据在采样频率、物理含义和寿命时间尺度上存在显著差异，本文不直接对异构原始信号进行端到端迁移，而是先分别构建具有物理意义的退化特征，再通过 Bearing Encoder 与 Flywheel Encoder 映射至共享 latent degradation space。随后引入 MMD、CORAL、DANN 和 AutoEncoder-joint 等域适配策略进行对比验证。实验表明，MMD 对齐可将附件 1 后 30% RUL 测试 MAE 从 154.2 天降低至 87.2 天，说明特征级深度迁移能够有效改善退化表征的跨域泛化能力。

避免这样写：

> 本文直接使用原始轴承振动波形和飞轮原始遥测数据完成端到端深度迁移训练。

因为这不是当前代码实际完成的内容。

## 7. 总结

当前项目使用的是“原始数据 -> 退化特征 -> 深度迁移训练”的路线。它比直接原始数据训练更稳定、更可解释，也更适合数模论文写作。区别在于，当前方法不是端到端从原始波形自动学习所有特征，而是在物理特征工程基础上进行深度 latent 对齐和迁移验证。

从任务完成角度看，这已经足以支撑“跨模态共享退化表征学习”和“迁移前后 RUL 误差对比”；从更高阶深度学习角度看，未来还可以扩展为原始信号级 CNN/Transformer 端到端迁移模型。
